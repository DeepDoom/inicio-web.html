Elemento HTML
(<a>)

Que es <a>

El elemento <a> Crea un enlace a otras páginas de internet, archivos o ubicaciones dentro de la misma página, direcciones de correo, o cualquier otra URL.

Uso:

Acepta cualquier elemento que acepte contenido de párrafo o cualquier elemento que acepte contenido de flujo, sin embargo siempre excluyendo los elementos <a> de acuerdo con el princípio lógico de simetria, si una etiqueta <a> como padre, no puede contener contenido interactivo, entonces el mismo contenido de <a> no puede tener una etiqueta <a> como su padre.

atributos:

Este elemento incluye los atributos globales.

Atributo Descripción
dowload Este atributo, indica descargar a los navegadores una URL en lugar de navegar hacia ella, por lo que el usuario será dirigido para salvarla como un archivo local. Si el atributo tiene un valor, éste se utilizara como nombre de archivo por defecto en el mensaje Guardar que se abre cuando el usuario hace clic en el enlace, sin embargo el usuario puede cambiar el nombre antes de guardar el archivo.
href Contiene una URL o un fragmento de URL al cual apuntar el enlace. Un fragmento de URL es un nombre "name" precedido por el simbolo de número #, el cual especifica una ubicación interna objetivo un ID de un elemento HTML dentro del documento. Las URLs no estánrestringidas sólo a documentos de internet basados en HTTP, sin embargo puede utilizar cualquier protocolo soportado por el navegador. Por ejemplo, file:, ftp: y mailto: funcionan en la mayoria de los navegadores.
hreflang Este atributo indica el lenguaje humano del recurso al que se enlaza. Este es únicamente informativo, sin ninguna funcionalidad incorporada. Los valores permitidos están determinados por BCP47.
referrerpolicy Indica que rferencia rederer enviar cuando la URL es recuperada:

"no-referrer" significa referer: el encabezado no será inviado.

"no-referrer-when-downgrade" significa sin referer: el encabezado será enviado cuando se navega a un origen sin HTTPS. Este es un comportamiento por defecto.

"origin" significa que el referrer estará em eñ progem origin de la página, no incluyente la información posteriror al dominio.

"origin-when-cross-origin" significa que la navegación a otros origenes será limitada al esquema, el host y el puerto, miestras que la navegación en el mismo origen incuirá la trayectoria de referencia.

"unsafe-url" significa que la referencia incuirá el origen y la trayectoria, pero no el fragmento, contraseña o nombre de usuario. Esto es inseguro, ya que puede filtrar datos desde una URL segura hacia URLs inseguras.
rel Especifica en donde desplegar la URL enlazadas. Es un nombre, o palabra clave, un contexto de navegación una pestaña, ventana, o <iframe>. Las siguientes palabras clave tienen significado especial.

_self: Carga la URL en el mismo contexto de navegación que el actual. Este es el comportamiento por defecto.

_blank: Carga la URL en un nuevo contexto de navegación. Usualmente es una pestaña, sin embargo, los usuarios pueden configurar los navegadores para utilizar para utilizar una ventana nueva en lugar de la pestaña.

_parent: Carga la URL en el contexto de navegación padre del actual. Si no existe el padre, este se comporta del mismo modo que _self.

_top: Carga la URL en el contexto más alto de navegación el cual es un ancestro del actual, y no tiene padre. Si no hay padre, este se comporta del mismo modo que _self.
type Especifica el tipo de medio en la forma de MIME para la URL enlazada. Esto es únicamente informativo, sin ninguna funcionalidad incorporada.
charset Este atributo define la codificación de caracteres de la URL enlazada. El valor debe de ser una lista delimitada por espacio y/o coma de caracteres definidos en RFC 2045. El valor por defecto es ISO-8859-1.
coords Para utilizar con el siguiente atributo shape, este atributo utiliza una lista de números separada por comas para definir las coordenadas del enlace en la página.
name Este atributo era requerido para anclas que definian una posible ubicación dentro de la página. En HTML 4.01, id y name podrian ser utilizados simultáneamente en un elemento <a> simpre y cuando tuvieran valores idénticos.
Eventos: onclik, ondblclick, onmousedown, onmouseup, onmouseover, onmousemove, onmouseout, onkeypress, onkeydown, onkeyup
